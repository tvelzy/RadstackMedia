Object export steps

  

The export objects wizard is used for exporting global objects only, apart from a particular website.

 

Step 1

 

In Step 1 of the Site export wizard, you can specify basic export settings:

 

File name

Name of the export package. A default name will be pre-filled and the package will be stored in <web project>\CMSSiteUtils\Export.

Do not preselect any objects

If chosen, no objects will be preselected in the next step.

Preselect all objects

If chosen, all site objects will be preselected in the next step.

Preselect objects changed after specific date

If chosen, only objects changed after the specified date will be selected in the next step.

Use previous export settings

If chosen, settings used in a previous export selected from the list below will be used.

 

Click the Next button to proceed to the next step.

 

Step 2

 

In Step 2 of the Export wizard, you can select which objects will be exported. The tree view on the left side of the screen shows particular categories. The Site category contains objects related to the selected website. The Global objects category contains global objects that can be used by all sites.

 

If you are exporting separate objects, only global objects will be offered here in the tree view. By selecting a category, a set of check boxes appears in the right part of the screen, letting you select which objects are to be exported and which are not. Preselection has been made based on your choice in Step 1.

 

After making all desired selections, click the Next button to proceed to the next step.

 

Step 3

 

A log appears, showing you the progress of export. You can abort exporting by clicking the Cancel button at any time. After the exporting is finished, a message appears at the top of the log, telling you the full path to the exported file. Click the Finish button. You will be redirected back to Site manager -> Sites.
